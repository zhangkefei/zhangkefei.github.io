用于前端代码规范的vscode插件

发表于 2018-11-12 更新于 2021-05-28 分类于 FrontEnd

本文列举一系列前端开发时，用于规范代码的 vscode 插件，及配置方法。

由于目前没有一个全能的插件可以将全部种类的前端代码格式化，因此采用的方法是结合使用多种插件，发挥各个插件的优势，最终达到接近团队代码规范的格式化效果。

下文以 HTML5,CSS3,ES6,Vue,stylus,TypeScript 技术栈为例，

安装插件

  • ESlint：javascript 代码检测工具，可以配置每次保存时格式化js。同时安装相关插件：vue-eslint-parser、babel-eslint、eslint-plugin-vue、eslint-plugin-html、eslint-plugin-import、eslint-plugin-node、eslint-plugin-promise
  • vetur：可以格式化 html、标准 css（有分号 、大括号的风格）、标准 js（有分号 、双引号的风格）、vue 文件。但是，格式化的标准 js 文件不符合 ESlint 规范，会加上双引号、分号等
  • Prettier - Code formatter：只关注格式化，并不具有 eslint 语法检查等能力，只关心格式化文件(最大长度、混合标签和空格、引用样式等)，包括 JavaScript、Flow、TypeScript、CSS、SCSS、Less、JSX、Vue、GraphQL、JSON、Markdown
  • Manta's Stylus Supremacy：格式化 stylus 的插件（如果用其他的 css 预处理器，可以探索其他插件使用方法），因为 vetur 会把 css 格式化有分号 、大括号的风格，此插件会把 css 格式化成 stylus 风格
